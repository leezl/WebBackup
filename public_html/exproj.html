Machine Learning for Attribute Recognition

In my work for my advisor in the robotics lab I am researching methods for learning attributes in images that are directly connected to words. That is, when given images that contain features like 'red', 'round', 'big', or 'shiny', I'd like to be able to automatically identify those traits. So it is related to feature extraction on images, but we are forcing the extracted features to represent traits that are human-understandable. We are currently using a combination of supervised and unsupervised learning techniques, both standard and recently developed in order to accomplish this goal.

DTW on Google N-Grams

A few semesters ago I took a Coursera class on Natural Language Processing, and implemented a very simple algorithm on Google N-Grams. Google N-Grams are a collection of scanned natural language documents such as books, newspapers, and legal records which have been scanned and documented into an n-grams. That is, for each word, the number of occurences overall and in each book, on each page, and per year have been counted, as well as for pairs, triples and quads of words. This has been shown to be a useful for many purposes. For example, documents are often labeled according to word occurences, and searches are based off of these labels. My project was attempting to show that important events could be described by finding associated words in the dataset. So given the phrase "World War" we would find that it peaks after 1950, and using this peak we can find words that peaked at the same time, to fidn related events and people, for example "soldier" and "ration." Using these words we could recreate blurbs about the events using standard sentence generation based around these words. Unfortunately I have not continued the project, and it needed improved database use, and speed improvements to run across so many words.

Ship Simulator

As a project for class my team built a ship simulator. The first was made using python Ogre for a game design class, the second was made usign Qt and OpenGl for a graphics class. Python Ogre provided many existing tools for quickly adding in sky boxes or spheres, as well as water and islands. However there were issues with config file pathing, especially when working on several machines. For the graphics class we attempted to handle that difficulty using Qt. We ended up hand coding many of the features already available in Python Ogre, and had trouble using OpenGl in Qt. If I can find my old code for either system I will make it available.
